Redroid环境搭建

发表于2025-12-19|更新于2025-12-19|移动安全
|浏览量:

拉取镜像

  • redroid/redroid:13.0.0_64only-latest // Android 13，纯 64 位
  • redroid/redroid:12.0.0_64only-latest // Android 12，纯 64 位
  • redroid/redroid:11.0.0-latest // Android 11，兼容 32 位

  1. redroid/redroid:11.0.0-latest是“多架构”镜像（manifest-list），官方同时提供了amd64和arm64两种变体，因此：在原生arm64主机上拉取时，会得到arm64版Android11； 在x86_64主机上拉取时，默认会得到amd64版本

  2. 要在x86_64宿主机上运行arm64架构的Android 11，需要显式指定平台并启用QEMU跨架构模拟

  3. 拉取redroid镜像

  4. 查看镜像是否拉取成功

启动容器

  • docker run: 创建并启动一个容器
  • -d: 启动后，把容器挂在后台，不进入
  • --platform-linux/arm64: 启动的是linux/arm64架构的容器
  • --privileged: 赋予容器和宿主机内核交互关键指令的权限，实现跨架构
  • -p 5555:5555: 端口映射（宿主机端口 : 容器端口）
  • --name Android-11: 给容器命名
  • redroid/redroid:11.0.0-latest: 指定根据哪个镜像文件创建容器

重启adbd

  1. 容器启动后，通过adb devices可能会发现设备offline，进入容器内重启adbd服务
